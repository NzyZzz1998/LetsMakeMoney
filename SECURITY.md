# 安全政策

## 支持范围

安全修复优先面向最新 Beta 与最近一个已发布 Beta。旧版本可能只提供升级建议。

## 报告漏洞

请使用 GitHub 仓库的 **Private vulnerability reporting** 私下报告。不要创建公开 Issue，也不要附带未脱敏的 `config.json`、`debug.log`、截图、用户名、绝对路径、证书或 token。

报告建议包含版本、Windows 版本、复现步骤、影响、最小脱敏日志和建议缓解方式。维护者会先确认收到，再评估严重度与修复计划。

## 范围

配置覆盖、任意路径访问、DLL/更新包替换、签名绕过、日志隐私、托盘/native 消息边界属于安全范围。单纯视觉问题或未造成安全影响的崩溃请使用 Bug Report。
