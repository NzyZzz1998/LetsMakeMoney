# LetsMakeMoney UI Polish Spec

**日期**：2026-07-08
**适用范围**：当前 v0.4 Beta 测试态与后续 UI 打磨
**原型入口**：`doc/prototypes/index.html`
**定位**：温暖桌面小挂件 / 橘猫金币小票便签风

---

## 1. 目标

当前 UI polish 的目标是把 LetsMakeMoney 从“能用的 Godot 桌宠工具”继续打磨为“愿意长期放在桌面的温暖小工具”。它不是主题系统，也不是后台管理页，而是一套默认视觉和交互语言：

> 小猫负责陪伴感，Panel 负责轻量薪资反馈，Settings 负责清楚可恢复的偏好设置。

本规格以当前实现和最新原型为准，不再按历史版本区分 UI 原型。

---

## 2. 设计关键词

- 温暖
- 轻量
- 奶油纸面
- 金币小票
- 橘猫陪伴
- 小而清楚
- 不像后台管理系统
- 不像调试工具

避免：

- 黑色玻璃面板。
- 冷蓝 SaaS 风。
- 系统默认菜单感。
- 过大的后台设置页。
- 大面积同色米黄。
- 控件套控件、卡片套卡片。
- 字体过大或整体拉伸导致的低清晰度。

## 4. 桌宠主界面

主界面由三部分组成：

1. 橘猫：视觉主体和交互入口。
2. Panel：薪资反馈小票。
3. 右键菜单 / 托盘：找回、设置、退出。

要求：

- 小猫和 Panel 同屏时风格统一。
- Panel 不遮挡小猫主体。
- 透明窗口空白区域尽量点击穿透。
- 缩放时优先重排真实控件尺寸和字号，不要把整块 UI 当图片拉伸。
- 小猫互动反馈要可见，但不能影响拖拽和右键操作。

---

## 5. Panel

### 5.1 折叠态

折叠态像小金币计数器：

```text
🟡 ¥35.28 工作中
```

要求：

- 背景使用奶油纸面，不使用黑色玻璃。
- 金额是主视觉，数字清晰。
- 状态为短文案：工作中 / 休息中 / 待机中 / 待设置。
- 不显示进度条和复杂详情。

### 5.2 展开态

展开态像小票/便签：

```text
今日已赚       ¥35.28
本月累计       ¥161.29
时薪           ¥7.41/小时
工作进度       58%
```

要求：

- 今日已赚层级最高。
- 本月、时薪、进度为详情。
- 进度条使用暖绿色到金币黄。
- 文字不挤压、不裁切。
- 悬停展开和离开收起动效轻微、可中断。

---

## 6. 右键菜单与托盘菜单

右键菜单与 Panel 使用同一套颜色、圆角、字体和阴影。

当前菜单结构：

```text
隐藏到托盘
设置
重新运行向导
窗口模式 >
  置顶悬浮
  融入桌面（实验）
选择宠物 >
  橘猫 v2
  橘猫 v1
  小猫旧素材
关于 LetsMakeMoney
退出
```

要求：

- 背景为奶油纸面。
- 文字为深咖啡。
- hover 使用淡金币黄。
- checked / 当前项使用金币黄或橘色强调。
- item 高度舒适但不笨重。
- 二级菜单靠右展开，不破坏点击穿透和右键命中。

托盘菜单保留显示/隐藏、设置、关于、退出等能力。托盘图标应使用当前 app icon，并针对 16 / 24 / 32 小尺寸继续检查清晰度。

---

## 7. Settings

Settings 当前方向是**紧凑暖色偏好设置面板**，不是大后台窗口。

结构：

```text
顶部：工资 / 桌宠 / 显示 / 面板 / 通用 tabs + 关闭按钮
中部：当前页标题 + section + row
底部：取消 / 保存 action bar
```

要求：

- 不再使用顶部大 header、返回按钮、搜索框和大面积 sidebar。
- 窗口尺寸控制在小偏好面板量级，目标约 680-720px 宽、520-540px 高。
- 每行只保留 label + 控件，具体设置项下方不常驻 helper 小字。
- 说明性内容放在底部低权重说明区，不能和真实设置项同层级。
- 不出现“row 外框 + 控件外框”的套圈感。
- 输入框、下拉框、slider、switch、button 使用统一 Warm Fluent Compact 风格。
- OptionButton popup 必须是暖色纸面风格，不使用深灰系统菜单。
- 保存反馈为轻量 inline message 或 toast，无内容时不占空间。

设置分类：

| 分类 | 主设置项 |
|------|----------|
| 工资 | 月薪、休息模式、上班时间、下班时间、每日工作小时数 |
| 桌宠 | 当前角色、素材/状态说明 |
| 显示 | 透明度、缩放、窗口模式、纯桌宠模式 |
| 面板 | 今日已赚、本月累计、时薪、工作进度、状态 |
| 通用 | Debug、开机自启、关闭隐藏到托盘、重置位置、恢复默认显示设置 |

---

## 8. Wizard

Wizard 与 Settings 使用同一套纸面、按钮、控件风格。

流程：

1. 欢迎。
2. 薪资。
3. 角色。
4. 确认。

要求：

- 不像 Godot 默认弹窗。
- 说明文字短。
- 输入控件清楚舒适。
- 下一步 / 上一步 / 完成按钮明确。
- 完成后保存配置并进入桌宠模式。

---

## 9. 图标

图标统一范围：

- exe 图标。
- 窗口图标。
- 托盘图标。
- 关于窗口图标。
- 文档和原型中的 icon 展示。

要求：

- 基于当前橘猫形象。
- 小尺寸优先清晰识别，不强求完整身体。
- 至少覆盖 16 / 24 / 32 / 48 / 64 / 128 / 256。
- 托盘图标需在 Windows 深色和浅色任务栏上复测。

---

## 10. 验收标准

- `doc/prototypes/index.html` 是当前唯一交互原型入口，不再按历史版本切屏。
- 原型覆盖桌宠、Panel、菜单/托盘、Settings、Wizard、Debug、发布包。
- Panel 和右键菜单明显属于同一套暖色设计系统。
- Settings 是紧凑偏好面板，不是后台表单。
- 所有关键按钮和控件在原型中都有能力说明。
- 文档中的 UI 描述与当前 Godot 实现方向一致。
- 后续 Godot UI 打磨以本规格和当前原型为准。
