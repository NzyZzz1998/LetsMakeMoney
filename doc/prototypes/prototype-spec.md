# LetsMakeMoney 当前交互原型说明

**日期**：2026-07-08
**原型入口**：`doc/prototypes/index.html`
**原型口径**：当前产品状态，不再按 v0.1 / v0.2 / v0.3 / v0.4 版本切分

---

## 1. 原型目标

本轮原型用于重新对齐 LetsMakeMoney 当前项目状态。旧原型主要表达版本演进，已经与当前 UI 打磨方向、Settings 收敛方向和功能现状产生偏差。因此新版原型改为按当前功能模块组织：

- 总览
- 桌宠主界面
- Panel 便签
- 右键菜单与托盘
- 设置面板
- 首次启动向导
- Debug 与验证
- 发布包

原型不是 Godot 场景实现文件，而是用于产品 review、工程对照、手动验证准备和后续 UI 打磨讨论的交互参考。

---

## 2. 当前产品状态摘要

| 模块 | 当前状态 |
|------|----------|
| 桌宠窗口 | Windows x86_64 下以透明无边框桌宠形态为目标，支持 native bridge、托盘、点击穿透和关闭隐藏到托盘 |
| 橘猫素材 | 默认接入橘猫 v2，保留旧素材回退；基础状态为 `idle` / `working` / `resting`，单击/双击/长按是基础状态上的互动延伸 |
| Panel | 折叠/展开可用，当前视觉方向为暖色金币小票/便签风 |
| 右键菜单 | 保留设置、重新运行向导、窗口模式、选择宠物、关于、退出；窗口模式和选择宠物使用二级入口 |
| 系统托盘 | 支持左键显示/隐藏、右键菜单、设置、关于、退出，是纯桌宠模式下的主要找回入口 |
| Settings | 正在从后台表单收敛为紧凑暖色偏好设置面板，包含工资、桌宠、显示、面板、通用五类 |
| Wizard | 首次启动和重新运行向导可用，覆盖欢迎、薪资、角色、确认流程 |
| Debug | 可通过配置开启大窗口、日志、输入状态和验证辅助 |
| 发布包 | v0.4 测试态以 zip 包思路维护，包含 exe、native dll、README、release notes、manifest/checksum |

## 3. 原型文件

```text
doc/prototypes/

```

`index.html` 是单文件 HTML 原型，可直接用浏览器打开。它引用项目现有素材，不复制资源：

- 橘猫 v2：`../../assets/pets/cat/orange_v2/...`
- 应用图标：`../../icons/app_icon_*.png`

## 4. 视觉方向

当前原型采用“温暖桌面小挂件 / 橘猫金币小票便签风”：

- 主体气质：可爱、温暖、轻量，像桌面上的小挂件。
- 主视觉元素：橘猫、金币、奶油纸面、小票/便签。
- 主色：奶油白、暖纸黄、金币黄、橘猫橙、深咖啡文字。
- 避免：黑色玻璃卡片、冷蓝 SaaS 风、系统默认菜单感、过重阴影、过度渐变。

设置页当前不再参考完整 Windows 11 后台式大窗口，而是收敛为更小、更聚拢的偏好设置面板。

## 5. 原型交互

### 5.1 全局导航

左侧功能导航切换当前模块。导航项不代表历史版本，只代表当前应用的功能区域。

### 5.2 快捷交互

| 按钮 | 行为 | 目的 |
|------|------|------|
| 展开/收起 Panel | 切换主界面 Panel 折叠/展开 | 验证 Panel 状态和视觉密度 |
| 切换工作状态 | 在工作中、休息中、待机中循环 | 验证金额、状态文案和小猫状态 |
| 模拟点击小猫 | 给小猫添加短暂点击反馈 | 验证单击/双击视觉反馈方向 |

### 5.3 桌宠主界面

可交互内容：

- 工作中 / 休息中 / 待机中按钮：切换小猫状态图和状态文案。
- 长按反馈：模拟小猫短暂互动反馈。
- 显示拖拽轨迹：说明拖拽后窗口位置保存到配置。

对应真实能力：

- 小猫 hover。
- 单击。
- 双击。
- 长按。
- 拖拽窗口。
- 右键菜单。
- 小猫与 Panel 之外的透明空白区域尽量点击穿透。

### 5.4 Panel 便签

可交互内容：

- 折叠态 / 展开态。
- 靠右边缘 / 靠底部预览。

对应真实能力：

- 鼠标悬停约 0.3 秒后展开。
- 鼠标移开约 0.5 秒后收起。
- 靠近屏幕右侧时优先向左展开。
- 靠近屏幕底部时优先向上展开。
- Settings 中可控制展开态显示项。

### 5.5 右键菜单与托盘

可交互内容：

- 切换窗口模式子菜单。
- 切换选择宠物子菜单。
- 收起子菜单。
- 悬停一级菜单项时展示对应子菜单。

菜单能力：

```text
隐藏到托盘
设置
重新运行向导
窗口模式 >
  置顶悬浮
  融入桌面（实验）
选择宠物 >
  橘猫 v2
  橘猫 v1
  小猫旧素材
关于 LetsMakeMoney
退出
```

托盘能力：

- 左键：显示/隐藏桌宠窗口。
- 右键：打开菜单。
- 退出：保存配置并结束进程。

### 5.6 设置面板

可交互内容：

- 工资 / 桌宠 / 显示 / 面板 / 通用五个中文 Tab。
- 保存 / 取消按钮。
- 下拉框、开关、滑杆、输入框视觉预览。

当前设置分类：

| 分类 | 内容 |
|------|------|
| 工资 | 月薪、休息模式、上班时间、下班时间、每日工作小时数 |
| 桌宠 | 当前角色、基础状态、互动动作 |
| 显示 | 透明度、缩放、窗口模式、纯桌宠模式、点击穿透说明 |
| 面板 | 今日已赚、本月累计、时薪、工作进度、状态 |
| 通用 | Debug 模式、开机自启、关闭时隐藏到托盘、重置窗口位置、恢复默认显示设置 |

说明性内容应降级展示，不应和真实设置项占同等视觉权重。

### 5.7 首次启动向导

可交互内容：

- 欢迎。
- 薪资。
- 角色。
- 完成。

真实流程：

1. 首次无配置时自动弹出。
2. 用户配置薪资、休息模式和工作时间。
3. 用户选择桌宠。
4. 保存配置并进入桌宠模式。
5. 后续可从右键菜单重新运行向导。

### 5.8 Debug 与验证

原型展示 Debug 模式的用途：

- 900x500 调试窗口。
- 输入/状态辅助观察。
- debug.log 事件分类。
- 自动验证脚本结果。

重点事件：

```text
clicked_single
clicked_double
clicked_hold
drag_start / drag_end
right_click_menu
```

### 5.9 发布包

原型展示当前发布包目标结构：

```text
LetsMakeMoney.exe
letsmakemoney_native.dll
README.md
v0.4-beta-notes.md
manifest / checksum
```

当前代码仍在 GitHub `test` 分支，尚未合并到 `main`。

## 7. 后续原型待打磨

- Settings 的实际 Godot 截图与原型仍需继续对齐。
- Wizard 需要补充更接近真实界面的输入控件细节。
- 右键菜单二级菜单在 Godot 中的 hover/定位还需要继续以实机为准。
- Panel 在不同 DPI、缩放、屏幕边缘的定位可继续补充更多状态。
- 托盘小图标需要在深色/浅色任务栏上继续做小尺寸清晰度检查。
