# LetsMakeMoney iOS v0.1 原型说明

## 定位

该原型承接 `doc/releases/ios-v0.1/prd.md`，用于确认 Apple 产品线首版的信息架构、跨设备分工、浅深色视觉和关键状态，不是可发布应用。

## 打开方式

直接在浏览器打开：

`doc/prototypes/ios-v0.1/index.html`

## 交互

- 顶部标签切换 iPhone、iPad 横屏、iPad 竖屏方案、系统入口、Apple Watch 和首次引导视图。
- 右上角半圆图标切换浅色与深色模式，并显示轻量反馈。
- iPhone 底部标签可切换“今日/日历”；齿轮按钮打开设置弹层，“调整今天”打开单日覆盖弹层。
- 设置弹层可切换工资、作息、系统三个分区；保存、取消、关闭、遮罩点击和 `Esc` 均有对应行为。
- iPad 侧栏可切换今日、日历和设置；“完整日历”和“调整今天”入口可继续操作。
- iPad 竖屏方案页可在 A/B/C 三种导航间切换，并在设备内实际切换今日、日历、返回和设置。
- Widget 与 Live Activity 可切换工作中、午休中、已完成、未配置四种状态，并可模拟启动/结束实时活动。
- Apple Watch 可切换剩余时间、今日收入、工作进度三种主指标，可模拟活动启停和离线降级。
- 首次引导可完整执行三步前进、返回、取消、校验和完成，并在确认页同步展示输入结果。
- 所有保存、取消、启停和异常操作均通过原型 Toast 给出即时反馈。

## 视图覆盖

### iPhone 今日

- 今日收入、工作状态、进度、本月累计和今日安排。
- 底部“今日/日历”双标签导航。
- 午休与真实工作日的产品说明。
- 设置与单日调整使用系统感弹层，不跳转到独立后台式页面。

### iPad 横屏

- B+C 组合：侧栏导航 + 双栏今日页。
- 左侧保留今日安排，右侧展示月历和月度摘要。
- 窄屏实现应切回 iPhone 式标签布局。

### iPad 竖屏方案

- 三种方案都明确禁止显示或唤出侧边栏，避免竖屏宽度被固定导航挤占。
- 最终确认采用 A：底部“今日/日历”双页签，与 iPhone 保持一致；两个核心页面均一步可达。
- B：顶部“今日/日历”分段切换，释放底部空间，形成更集中、更偏 iPad 工具的内容区。
- C：首页仅展示今日收入、进度和安排，通过“查看本月日历”下钻到日历，再显式返回今日。
- 三种方案均保留右上角设置入口、“调整今天”入口、今日安排和完整日历，确保对比只聚焦导航结构。
- A 的今日收入模块与今日安排模块使用同一顶部基线，不允许因组件默认外边距产生上下错位。
- B/C 仅保留为设计对照，不进入 iOS v0.1 实现范围。

### Widget 与 Live Activity

- 桌面 Widget。
- 工作中的 Live Activity。
- 午休中隐藏金额、显示复工倒计时的 Live Activity。

### Apple Watch

- 默认剩余时间主指标。
- 今日收入、今日安排和启动活动入口。
- 离线、指标切换、复杂功能与 Smart Stack 的文字说明。
- 离线状态会禁用启动/结束活动，恢复连接后重新开放操作。

### 首次引导

- 顶部仅保留中文标题和无文字进度，不展示 `onboarding.step.*` 等内部本地化键；实现时标题、进度、滚动内容和底部操作区必须在横屏键盘下保持稳定。
- 第一步配置月薪、币种和休息制度。月薪默认 `0.00`，零值首次聚焦清空，支持整数和最多两位小数。
- 选择大小周时直接询问“本周是大周还是小周”：大周代表本周周六上班，小周代表本周周六休息，不要求用户理解锚点日期。
- 第二步先输入上班时间和午休时长，立即按 8 小时有效工作推算午休结束、下班时间和有效工时。
- “调整详细时间”在同页展开专用时间控件；修改午休开始或结束时保持固定午休时长并双向联动，修改上下班后重新计算有效工时。
- 第三步确认月薪、休息制度、工作时间和午休结果；月薪或时间无效时阻止前进并显示错误，取消不会保存，完成后显示成功反馈并复位原型。

## 视觉规范

- 使用原生生产力结构和克制暖色强调。
- 浅色以暖白、深咖啡、金币黄和柔和绿色为主。
- 深色使用暖黑、暖灰和低饱和金色，不做简单反色。
- 金额使用等宽数字观感。
- 组件保持 Apple 平台的紧凑尺寸，不沿用 Windows 桌宠设置窗口结构。

## 原型边界

- 不表达系统实际刷新频率。
- 不包含加班、配置口令、同步、宠物和主题系统。
- 不作为 App Store 截图或营销页面。
- 实现需要遵循系统 Widget、Live Activity、灵动岛和 Watch 复杂功能的真实尺寸与交互限制。

## 2026-07-14 增量验证

- 浏览器：本机 Microsoft Edge Headless。
- 交互路径：月薪零值首次聚焦清空 -> 输入 `12000.25` -> 选择大小周/大周 -> 上班改为 `09:00` -> 午休改为 1 小时 -> 午休开始/结束双向联动 -> 下班改为 `19:00` -> 确认页。
- 结果：确认页显示月薪 `¥ 12,000.25`、`大小周 · 本周大周`、工作时间 `09:00-19:00`、午休 `13:00-14:00`；有效工时重算为 9 小时。
- 布局：390、700、1440 像素宽度均无横向溢出。
- 运行：浏览器 `pageerror` 为 0，未出现 `onboarding.step.*` 原始本地化键。
