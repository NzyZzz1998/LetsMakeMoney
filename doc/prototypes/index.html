版本内验证链路

M0事实同步current、原型、版本、分支、tag 与发布包一致P0
M1可信日志与脚本阻塞错误必须非零退出，普通模式语义可观测P0
M2托盘与诊断外部 PostMessage 走 native 回调，无正式包测试入口P1
M3有限体验精修有截图或日志证据才修改，无证据可零代码收口P2
M4高信任路径自启动、损坏配置、恢复默认先验证，失败才修P1

自动与人工边界

自动：脚本与日志检查退出码、错误文本、窗口显隐、任务栏样式、配置与注册表。
人工：真实 Windows通知区左键、注销/重启自启、2K 显示清晰度和真实菜单观感。
失败：进入 bugfix保留原始证据，不通过扩大 allowlist 或测试入口绕开。
通过：发布收口同步 verification、status、release notes、changelog 和包体。
Parser Error、Script Error、Invalid call 等任一出现即失败。

共享控件状态规范

默认
纸面底色、暖棕文字和轻边框，不抢占内容层级。
悬停
仅轻微提亮金币色，并保持文字和按钮尺寸稳定。
按下
亮度和位移反馈克制，不引起布局跳动。
禁用
降低饱和度但保留可读性，不使用深灰系统样式。
聚焦
使用单层金币色焦点环，不增加第二层容器边框。
错误
柔和红棕边框配合就地反馈，用户输入保持不变。
