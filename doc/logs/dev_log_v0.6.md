## 开发记录

### 2026-07-10 / V06-M0 事实、版本与开发基线

- 在 `project.godot` 建立 `application/config/version="0.6-beta"` 唯一版本源，并同步 Windows 导出版本元数据。
- 增加 Godot 与 PowerShell 版本读取 helper，About 和启动日志不再硬编码旧版本。
- 建立 v0.6 README、status、verification、release checklist，并将 `doc/current.md` 切换为 v0.6 实现态。
- 扩展文档状态扫描到 v0.6；`verify_v06.ps1 -StaticOnly` 与 `check_docs_status.ps1` 作为 M0 门禁。
- 技术决策：大历史文档和 v0.5 文档保持原位，只更新当前事实入口，避免破坏历史引用。

### 2026-07-10 / V06-M1 日志治理与可信验证

- 增加 info/debug/error 日志接口，保留 `write_boot_log` 兼容入口；日志达到 2 MB 后轮换为唯一 `debug.log.1`。
- 日志目录和轮换失败采用非阻塞 warning，不影响桌宠主流程。
- 普通模式不再保存单击/双击截图；Debug 或隔离验证环境变量可显式启用。
- 新增 `verification_common.ps1`，统一检查进程退出码和 Parser Error、Invalid Call、空实例等阻塞输出。
- v0.4、v0.5、v0.6、M4 与 M5 活跃入口接入统一判定；修复旧 v0.5 脚本“红错但返回通过”的假阳性。
- 验证证据：v0.6、v0.5、M4 正常通过；注入 `SCRIPT ERROR` 被拒绝并返回非零。

### 2026-07-10 / V06-M2 轻量诊断与脱敏摘要

- Settings 通用页新增“打开应用数据目录”和“复制诊断摘要”两个轻量入口。
- 摘要只包含版本、系统、非敏感运行配置、native 能力、日志元数据和最近语义事件；日志尾部读取限制为 64 KB。
- 交互截图只统计文件数量和总字节数，不读取文件名或图像内容；不生成 txt、Zip，也不上传数据。
- 剪贴板写入后必须完整读回一致才显示成功，打开目录和复制失败均提供可读反馈与 error 日志。
- 反馈默认隐藏，显示后 2.6 秒自动收起。
- 五次隔离验证均未泄露测试薪资、窗口坐标或字段名，也未产生 txt/Zip。配置恢复单次提示待 V06-M3 提供上游状态。

### 2026-07-10 / V06-M3 Settings/Wizard 事务与配置恢复

- Config 保存改为“临时文件写入与读回校验 -> 旧文件暂存 -> 原子替换”，失败时保留旧配置且清理临时文件。
- 损坏 JSON 会重命名为带时间戳的 invalid 备份，运行态加载默认值，并向 Settings 提供一次性恢复提示。
- Settings 保存先校验宠物和自启动能力，再持久化 Config，最后应用宠物与注册表；外部应用失败会持久化旧快照并恢复宠物、注册表和窗口状态。
- 无变化保存、取消与右上关闭不触发外部副作用；恢复默认仅重置既定显示/窗口字段，失败时恢复旧状态且不显示成功。
- Wizard 保存进入前 Config 与宠物快照；预览不再默认切换第一只宠物，取消、关闭和完成失败均恢复进入前状态。
- `verify_v06_config.ps1` 已验证安全保存、失败不破坏旧文件、损坏配置备份、单次提示以及事务调用顺序。

### 2026-07-10 / V06-M4 托盘、菜单与点击穿透边界

- 新增外部 `verify_v06_tray.ps1`，从独立测试进程枚举 `LetsMakeMoneyTrayMessageWindow` 并发送真实 `WM_APP + 101` 托盘回调消息；正式包没有新增测试入口。
- normal 与 pure pet 两种模式各完成 10 轮隐藏/恢复，逐轮检查窗口可见性、Win32 扩展样式、托盘请求/结果和窗口策略日志。
- 测试配置位于隔离 APPDATA，脚本无论成功失败都会结束自己启动的进程并恢复环境变量。
- 两类菜单无需业务修改：桌宠右键保留向导、窗口模式和宠物二级菜单；原生托盘保持显隐、设置、关于和退出。
- Godot Popup/Modal 的穿透暂停与恢复继续成对记录；原生托盘菜单未接入 Godot popup 生命周期，也未修改 native C++。
- 真实测试全部通过，因此未创建条件性 native bugfix。

## 关键决策

| 决策 | 背景 | 取舍 | 影响范围 | 后续观察 |
|---|---|---|---|---|
| 版本唯一来源使用 `application/config/version` | About、manifest 和文档曾分散硬编码 | 统一机器可读来源，打包与运行时共同读取 | project、Godot、PowerShell、文档 | 检查包内版本一致性 |
| 诊断只打开目录和复制脱敏摘要 | “诊断包”容易扩成文件导出和支持系统 | 不导出 txt/Zip，不上传 | Settings、日志、隐私 | 抽检摘要字段 |
| 剪贴板成功必须读回一致 | 历史上出现过失败却显示成功 | 保守失败优先于虚假成功 | Settings 反馈、日志 | Windows 剪贴板可用性 |
| 保存采用可补偿事务 | Config、注册表和宠物存在跨模块副作用 | 失败恢复全部状态，保留表单输入 | Settings、Wizard、Config、注册表、Pet | 补偿失败阻塞发布 |
| 原生托盘菜单不进入 Godot Popup 生命周期 | native 菜单由 Windows 消息窗口同步显示 | 不新增 C++ 菜单开关信号 | tray、passthrough、native | 命令执行后策略仍需日志验证 |
| UI 精修必须有证据编号 | 历史 UI 迭代容易范围蔓延 | 无证据允许零代码收口 | Panel、Pet、菜单、Settings、Wizard | 前后截图完整性 |
