# LetsMakeMoney v0.7 Bugfix Log

## V07-BUG-001 Settings 保存失败反馈不可见

- **发现阶段**：v0.7 最终候选 Acceptance
- **严重度**：发布阻塞
- **候选身份**：`main` / `9e76435973b8e12b3fe10199660fcb257aa968a6`
- **复现**：保持 Settings 工资页打开，将 `%APPDATA%\LetsMakeMoney\config.json.tmp` 制造成目录；月薪从 12000 改为 13000 后点击保存。
- **实际结果**：输入 13000 保留，原配置 SHA256 不变，日志记录 `config_save_failed` 与 `settings_save_failed`；0.35 秒和约 1.25 秒截图均无可见失败反馈。
- **预期结果**：页面必须显示可读失败提示，持续时间足够截图或人工识别；不得显示成功。
- **证据**：`.tmp_acceptance/v0.7-final-20260711-220351/evidence/19-settings-save-failed.png`、`20-settings-save-failed-delayed.png`、`save-failure-before-hash.txt`、`portable-debug-final.log`。
- **最小修复方向**：检查反馈容器在工资页的布局层级、可见区域和 action bar 遮挡；不改变现有事务回滚、输入保留和日志语义。
- **根因**：反馈容器作为独立 VBox 行插在固定高度 Settings Shell 的内容区和 action bar 之间；显示时增加纵向最小尺寸，导致紧凑窗口中反馈被推到可视边界之外。
- **修复**：将反馈容器移入固定底部 `ActionRow`，收紧内边距；保存事务、错误原因、输入保留、自动隐藏时间和配置回滚逻辑保持不变。
- **自动验证**：新增运行态 UI 合同，要求反馈可见、父节点为 `ActionRow` 且全局矩形位于 `SettingsShell` 内；修复前按预期失败，修复后通过。v0.7、v0.5、v0.4 与包验证通过。
- **真实复验**：新候选 Zip SHA256 `205C60CAF2B42A6B19EE2249D5623752A9F1624CF5B3ECE1DE3473B2BA5B74D8`；输入 `15322` 后制造 `config.json.tmp` 目录，0.6 秒和 1.5 秒稳定观察点均显示失败提示，原配置 SHA256 前后相同，失败日志完整。
- **证据**：`.tmp_acceptance/v0.7-bugfix-20260711-235257/evidence/`。
- **状态**：已修复，定向 Acceptance 通过。

## V07-OBS-001 自定义卸载确认页不遵循静默模式

- **发现阶段**：v0.7 Acceptance 环境恢复。
- **现象**：`unins000.exe /VERYSILENT /SUPPRESSMSGBOXES /NORESTART` 仍显示“卸载 LetsMakeMoney”自定义确认页并返回失败；取消后再次看到窗口，是环境恢复命令启动了第二个卸载实例，并非应用自发重启。
- **代码证据**：`installer/LetsMakeMoney.iss` 的 `InitializeUninstall()` 无条件创建并显示 `CreateCustomForm`，未检查 `UninstallSilent`。
- **工具边界**：Computer Use 直接按 `unins000.exe` 启动时可能误匹配其他软件的同名卸载器；必须用 PowerShell 绝对路径启动，再按窗口标题操作。
- **实际清理**：先按默认保留数据完成 GUI 卸载，程序目录、卸载注册项和进程清除，APPDATA 配置保留；随后使用隔离测试数据完成显式删除数据链路，二次确认后安装目录和隔离 APPDATA 清除，最终恢复原用户配置。
- **证据**：`.tmp_acceptance/v0.7-delete-data-20260712-203736/evidence/`。
- **严重度**：低；影响自动化/静默卸载，不影响普通交互卸载。后续如支持静默卸载，应在 silent 模式下默认保留数据并跳过自定义页。
- **状态**：已记录，当前不扩展修复范围。
