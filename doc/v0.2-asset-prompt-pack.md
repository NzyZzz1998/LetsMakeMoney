# LetsMakeMoney v0.2 小猫素材提示词执行集

**日期**: 2026-06-28  
**用途**: 配合 `doc/v0.2-asset-spike.md` 执行第一轮小猫动画素材 Spike。  
**原则**: 先确认单一角色设计，再生成三组最小动作；不要一开始生成完整 sprite sheet。

---

## 1. 执行顺序

第一轮只做四步：

1. 生成小猫基准形象。
2. 基于同一只小猫生成 `idle` 4 帧。
3. 基于同一只小猫生成 `working` 4 帧。
4. 基于同一只小猫生成 `resting` 2 帧。

每一步都先评估，不通过就回退到上一步，不继续扩动作。

---

## 2. 第一张图：小猫基准形象

### 2.1 目标

这张图用于确定最终角色视觉，不直接作为动画帧接入。

必须满足：

- 缩小到 64x64 后仍能看清轮廓。
- 角色正面或轻微 3/4 视角。
- 线条干净，色块简单。
- 没有背景、文字、水印、复杂道具。
- 看起来适合“陪用户赚钱”的桌面宠物。

### 2.2 提示词

```text
Use case: game assets
Asset type: Godot 2D desktop pet character reference
Primary request: create one small cute cat character for a desktop pet app named LetsMakeMoney, a cozy companion that stays on the user's desktop while they work and earn money.
Subject: one full-body cat with a rounded body, expressive face, tiny paws, soft ears, and a simple friendly silhouette. Add only one subtle office-themed detail if useful, such as a tiny collar tag shaped like a coin, but keep the design very simple.
Style/medium: flat cartoon vector-like illustration, clean soft outline, warm bright colors, low detail, readable at 64x64 pixels.
Composition/framing: centered full body, front-facing or slight 3/4 view, generous padding, no crop, neutral pose suitable as the base design for later animation frames.
Lighting/mood: cheerful, cozy, calm, gentle desk companion feeling.
Color palette: warm cream, light orange, soft brown, small yellow accent allowed.
Constraints: same character design must be reusable for animation frames; simple shapes; clear silhouette; no background; no text; no watermark.
Avoid: realistic fur, pixel art, complex background, dramatic lighting, heavy gradients, thin details, detailed clothing, changing accessories.
```

### 2.3 通过条件

| 检查项 | 通过标准 | 结果 |
|--------|----------|------|
| 缩小可读 | 64x64 仍能认出小猫 | 待填 |
| 风格匹配 | 扁平卡通，温暖治愈 | 待填 |
| 动画可复用 | 姿态简单，可扩展动作 | 待填 |
| 干净程度 | 无文字、水印、复杂背景 | 待填 |

## 3. 第二步：`idle` 4 帧

### 3.1 动作目标

`idle` 是默认待机动画。动作要小，但肉眼能看出生命感。

建议帧意图：

| 帧 | 意图 |
|----|------|
| `idle_000` | 基准站立/坐姿 |
| `idle_001` | 身体轻微上浮或胸口鼓起 |
| `idle_002` | 回到基准，尾巴位置略变 |
| `idle_003` | 身体轻微下沉或眨眼 |

### 3.2 提示词

```text
Use case: game assets
Asset type: Godot 2D AnimatedSprite2D frame sequence
Primary request: create 4 separate animation frames for the same flat cartoon cat idle loop, based on the approved reference cat design.
Subject: the exact same cute cat character, same colors, same proportions, same outline style, standing or sitting calmly.
Action: subtle breathing loop with a tiny body rise and fall, optional tiny tail sway, optional blink in one frame. The action must be visible but gentle.
Style/medium: flat cartoon vector-like illustration, clean soft outline, warm bright colors, low detail, readable at 64x64 pixels.
Composition/framing: four frames arranged left to right on a clean sheet or provided as separate frames; each frame centered on the same 64x64-friendly canvas; same scale; same camera; same baseline.
Frame notes: frame 1 neutral, frame 2 slight inhale/up, frame 3 neutral with tail shifted, frame 4 slight exhale/down or blink.
Constraints: frames must keep the same character identity, colors, proportions, eye style, and accessory details; no background; no text; no watermark; no new props.
Avoid: large pose changes, frame-to-frame color drift, changing head size, changing tail length, realistic fur, pixel art, complex shading.
```

### 3.3 文件命名

```text
assets/pets/cat/frames/idle_000.png
assets/pets/cat/frames/idle_001.png
assets/pets/cat/frames/idle_002.png
assets/pets/cat/frames/idle_003.png
```

### 3.4 通过条件

| 检查项 | 通过标准 | 结果 |
|--------|----------|------|
| 一致性 | 四帧看起来是同一只小猫 | 待填 |
| 动作幅度 | 呼吸/眨眼可见但不夸张 | 待填 |
| 对齐 | 脚底或身体中心基本稳定 | 待填 |
| 接入价值 | 值得放进 SpriteFrames 测试 | 待填 |

## 4. 第三步：`working` 4 帧

### 4.1 动作目标

`working` 是项目最重要的视觉表达：小猫在用户工作时间努力敲键盘。

动作必须比 `idle` 明显，且键盘不能漂移。

建议帧意图：

| 帧 | 意图 |
|----|------|
| `working_000` | 双爪准备，身体中位 |
| `working_001` | 左爪按下，右爪抬起 |
| `working_002` | 双爪回中，身体轻微弹起 |
| `working_003` | 右爪按下，左爪抬起 |

### 4.2 提示词

```text
Use case: game assets
Asset type: Godot 2D AnimatedSprite2D frame sequence
Primary request: create 4 separate animation frames for the same flat cartoon cat working loop, based on the approved reference cat design.
Subject: the exact same cute cat character sitting at a tiny simple keyboard.
Action: fast keyboard typing motion with alternating paws; the left paw and right paw press keys in alternating frames; the cat body bounces subtly with focused cheerful energy.
Style/medium: flat cartoon vector-like illustration, clean soft outline, warm bright colors, low detail, readable at 64x64 pixels.
Composition/framing: four frames arranged left to right on a clean sheet or provided as separate frames; each frame centered on the same 64x64-friendly canvas; same scale; same camera; same baseline. The keyboard remains in exactly the same position across frames.
Frame notes: frame 1 ready pose, frame 2 left paw down, frame 3 both paws near center, frame 4 right paw down.
Constraints: same character identity, colors, proportions, eye style, and accessory details; keyboard position must not drift; no screen text; no background; no watermark.
Avoid: laptop screen, detailed desk, extra office objects, changing keyboard shape, changing character design, realistic lighting, pixel art.
```

### 4.3 文件命名

```text
assets/pets/cat/frames/working_000.png
assets/pets/cat/frames/working_001.png
assets/pets/cat/frames/working_002.png
assets/pets/cat/frames/working_003.png
```

### 4.4 通过条件

| 检查项 | 通过标准 | 结果 |
|--------|----------|------|
| 敲键盘可读 | 缩小后仍能看出 typing | 待填 |
| 键盘稳定 | 键盘不随帧漂移 | 待填 |
| 小猫一致 | 颜色、头身比例不乱变 | 待填 |
| 循环顺畅 | 1-2-3-4 循环不突兀 | 待填 |

## 5. 第四步：`resting` 2 帧

### 5.1 动作目标

`resting` 用于非工作时间。它应该安静、治愈、低干扰。

建议帧意图：

| 帧 | 意图 |
|----|------|
| `resting_000` | 闭眼打盹，身体自然放松 |
| `resting_001` | 身体轻微起伏，睡意更明显 |

### 5.2 提示词

```text
Use case: game assets
Asset type: Godot 2D AnimatedSprite2D frame sequence
Primary request: create 2 separate animation frames for the same flat cartoon cat resting loop, based on the approved reference cat design.
Subject: the exact same cute cat character, curled up or sitting with eyes closed.
Action: sleepy breathing motion with a very gentle body rise and fall. Calm, quiet, non-distracting.
Style/medium: flat cartoon vector-like illustration, clean soft outline, warm bright colors, low detail, readable at 64x64 pixels.
Composition/framing: two frames arranged left to right on a clean sheet or provided as separate frames; each frame centered on the same 64x64-friendly canvas; same scale; same camera; same baseline.
Frame notes: frame 1 relaxed sleeping pose, frame 2 subtle breathing change with same silhouette and position.
Constraints: same character identity, colors, proportions, eye style, and accessory details; no background; no text; no watermark; no props unless identical in both frames.
Avoid: complex bedding, large body deformation, changed tail length, changed face shape, realistic fur, dark sleepy atmosphere, pixel art.
```

### 5.3 文件命名

```text
assets/pets/cat/frames/resting_000.png
assets/pets/cat/frames/resting_001.png
```

### 5.4 通过条件

| 检查项 | 通过标准 | 结果 |
|--------|----------|------|
| 休息感 | 一眼能看出睡觉/打盹 | 待填 |
| 不干扰 | 动作温和，不抢注意力 | 待填 |
| 循环自然 | 两帧来回不抽搐 | 待填 |
| 小猫一致 | 与基准形象一致 | 待填 |

## 6. 第一轮决策表

完成三动作后填写：

| 项 | 结论 |
|----|------|
| 是否继续补 `hover` / `clicked_*` | 待填 |
| 是否接入 `cat_sprite_frames.tres` | 待填 |
| 是否继续使用当前兔子占位 | 待填 |
| 是否需要换工具或人工绘制 | 待填 |

推荐决策规则：

- 如果 `idle` 过关但 `working` 不过关：保留基准形象，重新生成 `working`。
- 如果三组动作角色明显不一致：不接入，记录失败原因。
- 如果三组动作都过关：先接入 Godot 验证，再补交互动效。
- 如果 64x64 不清楚：回到基准形象，减少细节、加粗轮廓、提高色块对比。

## 7. Godot 接入检查

接入前确认：

```text
assets/pets/cat/frames/idle_000.png
assets/pets/cat/frames/idle_001.png
assets/pets/cat/frames/idle_002.png
assets/pets/cat/frames/idle_003.png
assets/pets/cat/frames/working_000.png
assets/pets/cat/frames/working_001.png
assets/pets/cat/frames/working_002.png
assets/pets/cat/frames/working_003.png
assets/pets/cat/frames/resting_000.png
assets/pets/cat/frames/resting_001.png
```

SpriteFrames 中动画名必须是：

```text
idle
working
resting
```

动画速度建议与 `PetResource.animation_speeds` 保持一致：

```text
idle: 2.0 fps
working: 0.8 fps
resting: 3.0 fps
```

注意：当前项目里的 `animation_speeds` 数值含义按现有实现走。实际观感如果太慢或太快，优先在 Godot 编辑器里调 SpriteFrames 动画速度，再同步回 `cat_resource.tres`。
