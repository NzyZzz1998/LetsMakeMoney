## 5. 优化方案与复测说明

本节用于记录上一轮手动验证中“不通过”的项目已经如何处理。上方表格中的“结果”列保留上一轮真实验证结果，不直接改写为通过；复测时请按照本节说明重新执行对应项目，并在原表格备注中补充新的结论。

本轮已辅助验证：
- `V03-MAN-044`：已通过本地窗口自动化打开右键菜单 > 设置，确认设置页现在是单层 Windows 窗口，标题为“设置”，内容直接铺满窗口，不再出现宿主窗口套内层设置弹窗。
- `V03-MAN-123 / V03-MAN-128`：已进入 Display 页，确认顶部“保存”按钮仍然可见；点击保存后设置页关闭，主窗口标题恢复为 `LetsMakeMoney`，桌宠状态恢复。
- `V03-MAN-121 / V03-MAN-122 / V03-MAN-123`：已恢复为下拉控件形态，并通过本地窗口自动化实际点击验证；`休息模式` 下拉能展开 `双休 / 单休`，`窗口模式` 下拉能展开 `置顶悬浮 / 融入桌面（实验）`。
- `V03-MAN-060 / V03-MAN-061 / V03-MAN-062`：已继续收紧小猫 native 点击穿透矩形和 Godot 碰撞形状，并降低 native 命中缓冲；Codex 已用 Win32 `WM_NCHITTEST` 做辅助验证，透明区域返回 `HTTRANSPARENT=-1`，猫身/Panel 核心返回 `HTCLIENT=1`。
- `V03-MAN-081`：发现小猫右键菜单存在偶发不弹出。根因是 v0.3 为修复点击穿透把小猫普通交互框收窄到核心身体区域，导致头部、耳朵、边缘等视觉上像小猫的位置不再进入右键输入路径。已新增右键专用上下文区域：普通左键 / hover / 拖拽仍使用小命中框，右键按下时 native 层临时放宽小猫命中范围，Main 再用 `get_pet_context_rect()` 弹出菜单。Codex 已用 Computer Use 右键点击旧小框外的猫头区域，菜单可弹出；同时用 Win32 `WM_NCHITTEST` 确认无按键状态下该区域仍返回 `HTTRANSPARENT=-1`，不影响普通点击穿透。
- `V03-MAN-022 / V03-MAN-023`：native 托盘左键事件已改为带状态的单次切换逻辑；保留 `WM_LBUTTONDOWN` 兜底，但会吞掉同一轮紧随其后的 `WM_LBUTTONUP / NIN_SELECT`，避免一次单击触发两次显隐。
