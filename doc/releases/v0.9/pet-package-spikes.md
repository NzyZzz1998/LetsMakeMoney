# v0.9 宠物包技术 Spike 结论

## SPIKE-001 通用宠物包与缓存

**结论：通过。**

- schema 主版本固定为 `1`，未知主版本直接拒绝。
- Classic Pro 与多多均使用同一 `PetPackageValidator` 和 `PetPackageImporter`，没有按 `pet_id` 特判。
- manifest 明确图集、逐帧毫秒时长、逻辑尺寸、pivot、脚底基线、命中策略、许可、来源、哈希和回退链。
- 缓存键由规范化 manifest 生成；版本、哈希或动作定义变化会失效。
- 旧 `.tres` 宠物继续由原扫描链加载；两个 Pro 包在 M4 只进行影子注册。
- 净化包只包含 `pet-package.json`、两个 WebP 图集、许可和来源说明，不含 QA、provenance、工作树路径或生产中间文件。

## SPIKE-002 动态轮廓

**结论：通过，但默认采用动作级 union。**

- Classic `working` 的 8 帧 alpha 矩形可全部提取，动作 union 能完整包围每帧。
- 本机 headless 基线中，8 帧首次扫描约 `12.8ms`；该成本只允许在导入/缓存阶段承担，不允许逐渲染帧重复扫描。
- 动作级 union 将 native region 数量从最多 8 组变化收敛为 1 组，优先保证拖拽、Panel 邻接和点击穿透稳定。
- M5 可消费预计算的逐帧矩形做视觉命中；native region 默认使用动作 union。若真实桌面证明 union 过宽，再以缓存后的逐帧区域做受控优化。
- 任意提取失败降级到逻辑显示矩形，但不得退回整窗拦截。

## SPIKE-003 指针跟随

**结论：通过，允许进入 M5 的受控实现。**

- 方向合同为顺时针 16 方向，每 22.5 度一个方向。
- 采样上限为 12.5Hz（80ms）；角度边界增加 4 度滞回；指针离开 250ms 后恢复基础动作。
- sleeping、拖拽、显式交互动作和 Modal 期间必须禁用指针跟随。
- 坐标转换、窗口缩放和 DPI 的真实桌面观感仍需在 M5/M7 用 Computer Use 与人工验收确认。

## M4 总结

M4 允许进入 M5。Classic 仍是影子候选，多多尚未进入用户列表；默认切换与正式可选门禁属于 M6。运行时素材许可继续服从仓库根目录 `ASSETS_LICENSE.md`，源码 MIT 许可不覆盖宠物视觉素材。
