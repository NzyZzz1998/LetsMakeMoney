## 1. DPI 与所有界面

分别把 Windows 缩放设置为 100%、125%、150%，每档重新启动候选 EXE：

1. 打开 Panel 折叠/展开、今日详情、右键一级/二级菜单、Settings 五页、Wizard 全流程。
2. 检查文字清晰、无裁切、无重叠、无黑角，控件不随动态文字改变布局。
3. 输入 0 元、普通金额和 9 位整数金额，确认 Panel 与今日详情不溢出。
4. 每个 DPI 保存完整窗口截图并记录编号 `V09-SHOT-001` 至 `007`。

通过标准：三档 DPI 的核心窗口均可读、可操作且与 v0.9 原型方向一致。

## 2. Classic 与多多动画

分别选择 Classic、多多，并在工作中、清醒休息、睡眠三个基础状态执行：

1. 在 working、awake_rest、sleeping 下各单击 5 次；动作必须符合当前状态，完整播放后恢复最新基础状态。
2. 双击 5 次，确认不会触发独立产品动作，也不会播放通用庆祝。
3. 长按 5 次，确认进入跑步准备；分别向左、向右和近垂直拖动，确认方向、窗口跟随和最近水平朝向稳定。
4. 释放拖动后确认播放收势或安全降级，并恢复最新基础状态；不得补发单击。
5. 在动作播放中再次输入，确认按优先级中断且不闪回错误帧。
6. 移动指针到猫的不同方向，确认 awake_rest 完整观察、working 短暂观察、sleeping 不观察，离开后不持续抖动。
7. 点击透明区域和可见轮廓边缘，确认动态命中区不过大、也不漏掉主体。
8. 打开右键菜单、Panel、Settings、Wizard，确认动画与点击穿透没有穿过模态层。

多多允许使用通用动作映射，但不得把同一静止帧伪装成专属动作完整。若差异不可辨认，记录为候选门禁失败并回退。

## 3. 选择、升级与回退

1. 使用旧 v0.8 配置启动，确认原宠物选择没有被自动改成 Classic。
2. 新建隔离配置，确认默认候选为 Classic。
3. 选择多多、退出并重启，确认选择持久化。
4. 点击“回滚到 v0.8 橘猫”，确认立即恢复且重启后保持。
5. 在测试副本中破坏目标包 manifest，确认按目标包 -> Classic -> v2 -> v1 -> 占位猫降级，日志记录原因。

## 4. Windows 专属回归

1. 普通模式：托盘左键隐藏/恢复，显示时任务栏存在入口。
2. 纯桌宠模式：托盘左键隐藏/恢复，显示时任务栏不得出现入口。
3. Settings、Wizard、菜单打开时点击穿透暂停，关闭后恢复。
4. Panel 靠近屏幕边缘时正确翻转，不遮挡猫或跑出工作区。
5. 今日详情单实例打开、记忆位置；把记录位置改到不可用屏幕后能安全回落。

## 5. 连续运行

候选包连续运行至少 2 小时，覆盖工作、午休或清醒休息中的至少两种状态。观察并记录：

- 环境动作是否自然触发且不过度重复；
- 动画是否卡死、长期停在交互帧或尺寸跳变；
- 命中区、拖拽、Panel 邻接、托盘找回和点击穿透是否仍正常；
- `debug.log` 是否出现持续错误、timeout 风暴或异常高频日志。
