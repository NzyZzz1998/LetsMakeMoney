## 1. 导入与启动

1. 删除 iPad 上旧的 `LetsMakeMoneyM3`/R9 测试项目，避免 Playgrounds 继续使用旧缓存。
2. 将 `LetsMakeMoneyM3R10-playgrounds.zip` 传到 iPad 的“文件”App。
3. 长按 Zip，选择“解压缩”。
4. 打开解压后的 `LetsMakeMoneyM3R10.swiftpm`；若系统询问打开方式，选择 Swift Playgrounds。
5. 等待首次解析和编译完成，点击运行。
6. 如果出现编译错误，不要改代码，拍下完整错误和文件名后停止验证。

### 已关闭的导入缺陷

- 首个候选包在 Swift Playgrounds 4.7 报错：`Could not read xcstrings file: stat(/xcstringstool): No such file or directory (2)`。
- 根因：iPad Playgrounds 环境缺少 String Catalog 编译工具，但旧导出包直接包含 `Localizable.xcstrings`。
- 修复：仅在 Playgrounds 导出时转换为 `zh-Hans.lproj/Localizable.strings`；正式工程继续保留 String Catalog。
- 复测要求：必须删除旧项目并使用本页所列新 SHA256 的 Zip 重新导入。

- 第二个候选包继续暴露四类 Apple SDK 类型错误：iOS 不可用的 `List(selection:)` 重载、动态 UIKit 颜色推断、只读引导步骤的 Binding 写入推断，以及不可用的 Live Region modifier。
- 修复：侧栏改为显式按钮选择；动态颜色与高对比度环境键改用 iOS 可用 API；引导步骤只读访问；反馈保留明确辅助功能标签但不再调用不可用 modifier。
- 上述兼容写法已加入 M3 静态合同；第三个候选包等待 iPad 复测。

- 后续候选包曾出现无错误详情的启动崩溃。通过 Core、资源、AppModel、页面和导航探针逐层二分，确认根因是 `Binding` 直接使用 `@MainActor` 实例方法引用：`set: model.select`。
- 修复为显式闭包 `set: { model.select($0) }` 后，最小导航探针和 R5 完整 App 均在 iPad 启动。该兼容规则已加入自动测试。
- 后续静默崩溃优先使用 `debugging-workflow.md` 与单一 Debug Hub；旧探针仅作为最后的最小二分工具。

## 判定规则

- `M3-MAN-001` 或 `M3-MAN-002` 未通过：M3 保持阻塞，先修 Apple 平台构建问题。
- 主路径失败、配置被错误覆盖或页面无法关闭：M3 不通过。
- 仅有轻微视觉差异但操作和数据正确：记录为非阻塞优化候选。
- R10 四项定向复测全部通过后，关闭 `IOS01-M3R-014`，再继续 M3 剩余收口。
