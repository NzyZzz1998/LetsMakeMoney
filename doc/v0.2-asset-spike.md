# LetsMakeMoney v0.2 素材 Spike

**版本**: v0.2 Beta 非阻塞素材探索  
**日期**: 2026-06-28  
**状态**: 待执行  
**目标**: 在不引入重型依赖的前提下，探索一套可接入 Godot SpriteFrames 的扁平卡通小猫动画素材流程。

---

## 1. 定位

本 Spike 只解决一个问题：当前 AI/免费素材是否能产出足够稳定的小猫动画，替换 v0.1 的兔子占位素材。

本 Spike 不阻塞 v0.2 Beta 主线。若素材质量不达标，继续使用现有占位素材，并把失败原因记录在本文档中。

v0.2 主线仍优先交付：

- 默认桌宠模式与 Debug 模式拆分
- 透明/紧凑桌宠窗口
- 角色区域交互
- 系统托盘
- 开机自启
- 设置体验打磨

## 2. 资源目标

### 2.1 角色

| 项 | 要求 |
|----|------|
| 角色 | 小猫 |
| 风格 | 扁平卡通，非像素风 |
| 线条 | 清晰、柔和、外轮廓稳定 |
| 色彩 | 鲜明温暖，低复杂度，适合桌面常驻 |
| 原生尺寸 | 64x64 px |
| 显示尺寸 | 约 120x120 px，项目内可继续按 50%-200% 缩放 |
| 背景 | 透明 PNG，或可通过纯色背景后处理抠透明 |
| 朝向 | 默认正面或轻微 3/4 视角，所有帧保持一致 |

### 2.2 优先动作

先做三动作最小闭环：

| 动画名 | 帧数 | 播放方式 | 周期 | 描述 |
|--------|------|----------|------|------|
| `idle` | 4 | 循环 | 2000ms | 站立呼吸、轻微上下起伏或尾巴轻摆 |
| `working` | 4 | 循环 | 800ms | 坐在小键盘前快速敲键盘 |
| `resting` | 2 | 循环 | 3000ms | 闭眼打盹、身体轻微起伏 |

三动作质量达标后，再补充交互动作：

| 动画名 | 帧数 | 播放方式 | 周期 | 描述 |
|--------|------|----------|------|------|
| `hover` | 1 | 静态 | - | 看向鼠标，轻微抬头或睁眼 |
| `clicked_single` | 3 | 单次 | 600ms | 被摸头、轻弹或开心眨眼 |
| `clicked_double` | 3 | 单次 | 800ms | 欢呼、跳起或小金币/花纸反馈 |
| `clicked_hold` | 2 | 循环 | 1000ms | 呆住、疑惑或眼神放空 |

- `cat_sprite_frames.tres` 与 `cat_resource.tres` 仍沿用当前项目架构。

## 4. 轻量工作流

### 4.1 第一阶段：生成关键姿态

先生成每个动作的关键姿态图，不急着一次生成完整 sprite sheet。

推荐顺序：

1. 生成统一小猫设计图。
2. 以同一只小猫为基准生成 `idle` 4 帧。
3. 生成 `working` 4 帧。
4. 生成 `resting` 2 帧。
5. 人工检查一致性。
6. 质量达标后再裁切为 64x64 PNG。

### 4.2 第二阶段：整理帧

每帧需要满足：

- 画布尺寸一致。
- 小猫脚底/身体中心大致对齐。
- 角色大小一致。
- 透明背景干净。
- 没有文字、水印、阴影地面或复杂背景。
- `working` 的键盘位置稳定，不能每帧漂移。

### 4.3 第三阶段：接入 Godot

在 Godot 编辑器中：

1. 打开 `assets/pets/cat/cat_sprite_frames.tres`。
2. 替换或新增 `idle` / `working` / `resting` 动画帧。
3. 保持动画名称与 `PetManager.state_to_anim_name()` 一致。
4. 保持 `cat_resource.tres` 引用当前 `cat_sprite_frames.tres`。
5. 运行主场景，验证工作时间和交互状态能正确切换动画。

## 6. 透明背景策略

优先使用透明 PNG。若生成工具不能稳定输出透明背景，则使用纯色背景后处理：

- 默认纯色背景：`#00ff00`
- 如果角色包含绿色，改用：`#ff00ff`
- 背景必须纯色，无阴影、无渐变、无地面、无反光。

抠图后检查：

- 四角透明。
- 角色边缘没有绿色/紫色描边。
- 半透明边缘不过度发虚。
- 缩小到 64x64 后仍清晰。

## 10. 当前建议

本项目当前最推荐的素材路线：

1. v0.2 只做 `idle` / `working` / `resting` 三动作 Spike。
2. 质量达标才替换 `cat_sprite_frames.tres`。
3. 质量不达标就只保留探索记录，继续推进桌宠窗口、托盘和开机自启。
4. v1.0 前再决定是否投入正式定制插画或专门动画工具。
