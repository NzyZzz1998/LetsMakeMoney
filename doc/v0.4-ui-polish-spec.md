# LetsMakeMoney v0.4 UI Polish Spec

**版本**: v0.4 Beta UI polish 规格
**日期**: 2026-07-05
**定位**: 极简生产力小工具 + 宠物陪伴
**状态**: 已确认方向，待进入原型与 Godot UI 实现

---

## 1. 目标

v0.4 UI polish 的目标不是为 LetsMakeMoney 增加主题系统，也不是把应用改成完整游戏化桌宠。它要解决的是当前界面“能用但气质不统一”的问题，让默认体验更像一个轻量、可信、低压力的桌面生产力小工具，同时保留橘猫带来的陪伴感。

本规格将 v0.4 默认视觉方向定义为：

> 极简生产力小工具 + 宠物陪伴。

具体含义：

- 小猫负责陪伴感、情绪反馈和桌面存在感。
- Panel 负责轻量薪资反馈，像贴在小猫旁边的薪资便签，而不是压迫式效率仪表盘。
- 设置窗口保持工具化、清晰、可恢复，不做可爱化主题包装。
- 主题系统、主题切换、主题商店和用户自定义外观不进入 v0.4。

---

## 2. 设计原则

### 2.1 信息轻，但不能失真

Panel 可以更柔和、更轻，但金额、工作状态、今日/本月/时薪/进度等核心信息不能被弱化成装饰。用户要能一眼确认“现在赚了多少、当前在工作还是休息”。

### 2.2 陪伴感来自小猫，不来自复杂 UI

UI 不抢小猫的戏。Panel、提示和设置都应该像辅助层，避免大面积高饱和色、过度动效、夸张圆角或玩具化按钮。

### 2.3 设置窗口继续偏工具化

设置承担配置、恢复、诊断和平台能力说明，应该稳定、清楚、可预期。它可以更整洁、更一致，但不应变成“宠物主题设置页”。

### 2.4 默认体验不是主题

v0.4 只统一默认体验，不提供外观切换入口。所有颜色、间距、动效和组件规格都以默认体验为准，后续主题系统作为独立版本规划。

## 3. 信息架构

### 3.1 桌宠主界面

主界面由三层组成：

| 层级 | 内容 | UI 角色 |
|------|------|---------|
| 第一层 | 橘猫 | 主体、陪伴、交互入口 |
| 第二层 | 薪资 Panel | 轻量信息反馈 |
| 第三层 | 右键菜单 / 托盘 / 设置入口 / 应用图标 | 找回、配置、退出、识别 |

默认视线应先落在小猫，再看到 Panel。Panel 不应遮挡小猫主体，也不应离小猫过远。

### 3.2 Panel 折叠态

折叠态采用“金额 + 短状态”：

```text
¥12.34  工作中
```

状态文案建议：

| 状态 | 文案 |
|------|------|
| working | 工作中 |
| resting | 休息中 |
| idle | 待机中 |
| unknown / 未配置 | 待配置 |

折叠态不显示进度条，不显示今日/本月/时薪等细节，避免常驻界面变成密集仪表盘。

### 3.3 Panel 展开态

展开态保留完整信息，但层级调整为：

1. 今日已赚：主信息。
2. 当前状态：辅助判断。
3. 本月累计、时薪、工作进度：详情信息。

展开态建议结构：

```text
今日已赚      ¥12.34
状态          工作中
本月累计      ¥1,234.56
时薪          ¥42.00 / 小时
工作进度      36%
```

进度条应保持低对比，不要变成视觉主角。

### 3.4 设置窗口

设置继续保留五个分类：

```text
Salary / Pet / Display / Panel / General
```

设置页职责：

| 分类 | 主要职责 |
|-----|----------|
| Salary | 月薪、单双休、工作时间 |
| Pet | 角色选择与宠物资源 |
| Display | 透明度、缩放、窗口模式、纯桌宠、点击穿透说明 |
| Panel | Panel 显示项 |
| General | Debug、开机自启、关闭隐藏到托盘、重置窗口位置、恢复默认 |

v0.4 设置窗口的整体风格参考 Windows 11 设置窗口：左侧分类导航、右侧卡片式内容、清晰标题层级、暗色 surface、柔和边界、稳定的搜索入口或搜索占位。该参考只用于布局和质感，不复制 Windows 系统设置的账号、设备、系统更新等无关内容。

设置窗口不引入 Theme Tab，不提供外观选择入口。

### 3.5 右键菜单

右键小猫菜单需要从“平铺所有快捷入口”改为更清晰的分层菜单：

```text
设置
窗口模式 >
  置顶悬浮
  融入桌面
选择宠物 >
  橘猫
关于 LetsMakeMoney
退出
```

要求：

- `窗口模式` 和 `选择宠物` 不再作为平铺快速入口。
- 鼠标悬停一级项时，右侧展开二级菜单。
- 当前窗口模式和当前宠物需要有勾选、高亮或 `当前` 标记。
- 只有一个宠物时，`选择宠物` 仍可展示当前宠物，但不可用项要禁用或标记为当前，不报错。
- 子菜单展开不能破坏右键菜单点击、点击穿透、拖拽恢复或托盘降级入口。

### 3.6 图标系统

v0.4 统一应用图标、窗口图标、托盘图标、导出 exe 图标和关于窗口图标。

要求：

- 图标基于当前橘猫形象，但小尺寸优先清晰识别，不强求展示完整身体。
- 至少准备或导出 16 / 24 / 32 / 48 / 64 / 128 / 256 尺寸。
- 16 / 24 尺寸允许使用简化猫头或轮廓版本，避免缩小后糊成一团。
- 托盘图标需同时检查 Windows 深色任务栏和浅色任务栏。
- 窗口图标、exe 图标和托盘图标保持同源识别，但允许托盘小尺寸单独优化。

## 4. 视觉系统

### 4.1 色彩

v0.4 使用低饱和、暖灰、墨色和少量琥珀强调色。

| Token | 建议值 | 用途 |
|-------|--------|------|
| `color.surface.panel` | `rgba(25, 24, 22, 0.78)` | Panel 背景 |
| `color.surface.panel-hover` | `rgba(32, 30, 27, 0.86)` | Panel hover / 展开背景 |
| `color.surface.settings` | `#20242E` | 设置窗口主体，接近 Windows 11 深色设置背景 |
| `color.surface.settings-sidebar` | `#171B24` | 设置窗口左侧导航 |
| `color.surface.settings-elevated` | `#2A2F3B` | 设置页内卡片和控件背景 |
| `color.surface.settings-hover` | `#333A48` | 设置项 hover / selected |
| `color.border.subtle` | `rgba(255, 255, 255, 0.12)` | Panel / 设置边线 |
| `color.text.primary` | `#F7F3EA` | Panel 主文字 |
| `color.text.secondary` | `#C8C0B2` | Panel 次级文字 |
| `color.text.settings` | `#EDEDED` | 设置主文字 |
| `color.text.muted` | `#A8A8A8` | 设置说明 |
| `color.accent.warm` | `#F2A23A` | 小面积强调、状态点 |
| `color.success` | `#7CCB8A` | 保存成功、可用状态 |
| `color.warning` | `#E6B450` | 需要注意、实验能力 |
| `color.danger` | `#E06C75` | 危险或失败 |

使用约束：

- Panel 不使用纯黑大底，避免在透明桌面上显得重。
- 设置窗口可以更暗更工具化，但要保证文字对比。
- 琥珀色只能作为强调，不作为大面积背景。
- 不使用紫蓝渐变、霓虹、玻璃拟态大 blur 或主题商店式外观。

### 4.2 字体与字号

Godot 中优先使用 Windows 系统中文字体，不在 v0.4 引入复杂字体依赖。当前实现优先使用 `Microsoft YaHei UI`，回退到 `Microsoft YaHei` 与 `Segoe UI`。Panel 缩放必须通过真实控件尺寸、间距和字号重排实现，不能直接整体拉伸包含文字的 Control，否则中文会变成低清贴图。

| 场景 | 字号 | 权重/样式 |
|------|------|-----------|
| Panel 折叠金额 | 24 基准，随缩放重算 | 中高权重，数字清晰 |
| Panel 折叠状态 | 15 基准，随缩放重算 | 次级文本，但必须可读 |
| Panel 展开主金额 | 32 基准，随缩放重算 | 主信息 |
| Panel 展开标题/详情 | 15-16 基准，随缩放重算 | 常规 |
| 设置导航 | 17 | 常规 |
| 设置表单标题 | 18 | 常规偏强 |
| 设置说明文字 | 15 | 次级，但不得低对比到不可读 |
| 设置按钮/输入控件 | 16 | 常规 |

数字使用等宽或 tabular 风格优先，避免金额刷新时宽度跳动。若 Godot 字体不支持 tabular figures，则通过固定宽度容器减少跳动。

### 4.3 圆角、边框与阴影

| 组件 | 圆角 | 边框 | 阴影 |
|------|------|------|------|
| Panel 折叠态 | 8px | 1px subtle | 无或极轻 |
| Panel 展开态 | 8px | 1px subtle | 极轻 |
| 设置窗口 | 8px | 1px subtle | 普通对话框阴影 |
| 设置卡片 | 6-8px | 1px subtle | 无或极轻 |
| 设置侧边导航项 | 4-6px | 无 | 无 |
| 设置按钮 | 4-6px | 根据状态 | 无 |
| 输入框/下拉框 | 4-6px | 1px | 无 |

不要使用过大的胶囊圆角，避免看起来像主题皮肤或玩具 UI。

### 4.4 间距

使用 4px / 8px 节奏：

| Token | 值 | 用途 |
|-------|----|------|
| `space.xs` | 4 | 紧密元素 |
| `space.sm` | 8 | 标签与控件间距 |
| `space.md` | 12 | Panel 内边距 |
| `space.lg` | 16 | 设置分组间距 |
| `space.xl` | 24 | 大分组或窗口边距 |

Panel 折叠态左右内边距建议 12-14px，上下内边距建议 8-10px。设置窗口外边距建议 16px，分组间距建议 12-16px。

## 5. Panel 组件规格

### 5.1 折叠态

目标：常驻、轻量、不打扰。

建议尺寸：

```text
宽度：160-190px
高度：48-56px
```

布局：

```text
┌──────────────────┐
│ ¥12.34   工作中 │
└──────────────────┘
```

要求：

- 金额垂直居中。
- 状态可以与金额同行，靠右或紧随金额。
- 状态文字不应抢金额层级。
- 金额过长时优先扩大折叠态宽度，其次缩小金额字号，最后才省略。
- 折叠态 hover 0.25-0.35 秒后展开。

### 5.2 展开态

目标：提供完整薪资信息，但仍像轻量便签。

建议尺寸：

```text
宽度：280-320px
高度：190-220px
```

布局：

```text
今日已赚          ¥12.34
状态              工作中

本月累计          ¥1,234.56
时薪              ¥42.00 / 小时
工作进度          36%
[进度条]
```

要求：

- 今日已赚放在最上方，视觉权重最高。
- 状态行紧跟主金额，帮助用户判断当前收益是否在增长。
- 进度条低对比，不使用高饱和色填满。
- 展开态不遮挡小猫主体。
- 鼠标移出 0.45-0.6 秒后收起，避免抖动。

### 5.3 Panel 状态

| 状态 | 视觉 |
|------|------|
| default | 半透明暖墨底，subtle border |
| hover | 背景略深，边框略亮 |
| expanded | 背景比折叠态更实，内容层级清晰 |
| disabled / 无薪资 | 金额为 `¥0.00`，状态为 `待配置` |
| error / 数据异常 | 状态显示短提示，不覆盖金额 |

### 5.4 Panel 与小猫关系

- 默认距离小猫 8-16px。
- Panel 不贴脸，不遮挡头部、尾巴和主要动作。
- 缩放变化后 Panel 与小猫距离按视觉比例调整。
- 点击穿透白名单应覆盖 Panel 可见区域，不应覆盖透明空白。

## 6. 设置窗口规格

### 6.1 视觉定位

设置窗口是工具面板，不是陪伴界面。v0.4 的视觉参考为 Windows 11 设置窗口，它应保持：

- 清晰。
- 稳定。
- 可恢复。
- 工程说明可理解。
- 左侧导航清楚。
- 右侧设置项像系统设置卡片一样可扫描。

不使用猫咪装饰、不使用大面积暖色背景、不加入主题化插画。

### 6.2 布局

建议结构：

```text
┌────────────────────────────────────────────────────┐
│ ← 设置              查找设置                 ×     │
├───────────────┬────────────────────────────────────┤
│ Salary        │ Display                            │
│ Pet           │ ┌────────────────────────────────┐ │
│ Display       │ │ 透明度                  80%   │ │
│ Panel         │ └────────────────────────────────┘ │
│ General       │ ┌────────────────────────────────┐ │
│               │ │ 窗口模式              置顶悬浮 │ │
│               │ └────────────────────────────────┘ │
├───────────────┴────────────────────────────────────┤
│ 保存状态                              取消  保存   │
└────────────────────────────────────────────────────┘
```

要求：

- 只呈现一个设置窗口，不出现大宿主窗口套小设置窗。
- 顶部标题、搜索入口、左侧分类导航、右侧内容、底部操作区层级清晰。
- 左侧导航项至少包含 Salary / Pet / Display / Panel / General。
- 右侧内容使用卡片式设置项，标题、说明和控件在同一行或同一卡片中保持关联。
- 保存按钮固定在底部或顶部一致位置，不在多个位置重复出现。
- 长说明文字自动换行，不撑破窗口。
- 需要重启或重新显示窗口生效的项，在控件附近说明。
- 如果 v0.4 暂不实现真实搜索，搜索框作为禁用或占位状态展示，不影响键盘焦点和设置操作。

### 6.3 表单控件

| 控件 | 用途 | 要求 |
|------|------|------|
| SpinBox | 月薪、时间、数值 | 宽度稳定，单位清晰 |
| OptionButton | 单双休、窗口模式 | 必须可点击，展开层级不被遮挡 |
| HSlider | 透明度、缩放 | 右侧显示百分比 |
| CheckButton | Debug、自启动、关闭隐藏、纯桌宠 | 文案明确风险和效果 |
| Button | 保存、取消、重置、恢复默认 | 主次层级清楚 |

单双休、窗口模式继续使用下拉形式，不用开关替代。

### 6.4 保存反馈

保存反馈分为四类：

| 场景 | 文案方向 | 视觉 |
|------|----------|------|
| 保存成功 | 已保存 | success 色，短暂显示 |
| 无变化 | 没有需要保存的修改 | muted 色，快速返回 |
| 保存失败 | 保存失败：原因 | danger 色，保留窗口 |
| 需要重启/重显 | 已保存，重新显示窗口后生效 | warning 色 |

反馈必须出现在设置窗口内，不使用会被桌宠窗口遮挡的临时浮层。

### 6.5 恢复类操作

`重置窗口位置` 和 `恢复默认设置` 必须区别对待：

- 重置窗口位置：轻操作，可直接执行，执行后提示结果。
- 恢复默认设置：影响范围更大，需要确认或明确说明不会清空薪资/角色数据。

## 6.6 右键菜单规格

### 6.6.1 菜单结构

右键小猫菜单使用一级命令 + 二级子菜单结构：

```text
设置
窗口模式 >
选择宠物 >
关于 LetsMakeMoney
退出
```

`窗口模式` 子菜单：

```text
✓ 置顶悬浮
  融入桌面
```

`选择宠物` 子菜单：

```text
✓ 橘猫
```

### 6.6.2 交互

- 鼠标移动到 `窗口模式` 或 `选择宠物` 时，子菜单在右侧展开。
- 鼠标离开父项和子菜单区域后，子菜单延迟短暂关闭，避免轻微移动导致菜单闪烁。
- 点击子菜单项后应用配置，并关闭菜单。
- 点击一级 `设置` 直接打开设置窗口。
- 点击 `退出` 保存配置并退出。
- `关于 LetsMakeMoney` 打开关于窗口。

### 6.6.3 视觉

- 菜单背景和设置窗口同属深色工具层，但菜单更紧凑。
- 当前项用勾选、左侧标记或高亮表达，不能只靠颜色。
- 子菜单右侧展开时不得超出屏幕；靠右边缘时允许向左展开。
- 菜单项高度建议 28-34px，保证鼠标操作舒适。

## 6.7 图标规格

### 6.7.1 尺寸

| 尺寸 | 用途 | 要求 |
|------|------|------|
| 16x16 | 托盘最小尺寸 | 使用简化轮廓，不能糊成色块 |
| 24x24 | 高 DPI 托盘 / 小控件 | 保留猫头识别度 |
| 32x32 | 窗口标题栏 / 小图标 | 边缘清晰 |
| 48x48 | Windows 常见图标 | 可展示猫头和少量身体特征 |
| 64x64 | 关于窗口 / release 文档 | 细节稳定 |
| 128x128 | 高 DPI 图标 | 保持圆润和品牌感 |
| 256x256 | exe / ico 源图 | 作为主源尺寸 |

### 6.7.2 风格

- 基于橘猫形象，优先使用猫头或猫头 + 胸口的简化构图。
- 小尺寸去掉复杂阴影和细线。
- 轮廓在深色、浅色任务栏上都要有足够对比。
- 不使用 emoji 作为结构图标。
- 不使用过细线条，避免托盘缩小时消失。

### 6.7.3 接入

- Godot export preset 使用最终 ico。
- Windows 原生托盘使用同源图标或小尺寸优化图标。
- 关于窗口和 release notes 使用高分辨率 PNG。
- 发布包 manifest 记录图标版本或资源路径。

## 7. 动效规格

### 7.1 UI 动效节奏

| 动效 | 时长 | 说明 |
|------|------|------|
| Panel hover 反馈 | 120-160ms | 背景/边框轻变化 |
| Panel 展开 | 180-240ms | scale / opacity，避免宽高直接硬跳 |
| Panel 收起 | 140-200ms | 比展开略快 |
| 保存反馈出现 | 120-160ms | opacity |
| 保存反馈消失 | 180-240ms | opacity |
| 设置 Tab 切换 | 0-120ms | 可轻微淡入，不做滑动大动画 |

要求：

- UI 动效不阻塞输入。
- Panel 展开/收起要可中断。
- 不用弹簧过强的 bounce。
- 小猫动画由动画模块负责，UI 不通过强烈缩放替代小猫动作。

### 7.2 降低干扰

用户没有主动交互时，Panel 不应频繁闪动。金额刷新不做明显跳动，只保持数字更新稳定。

---

## 8. 文案规范

整体文案应该像一个安静的小工具：

- 简洁。
- 不卖萌过度。
- 不使用工程黑话直面普通用户。
- 风险和限制说清楚。

示例：

| 场景 | 推荐文案 | 避免 |
|------|----------|------|
| 工作状态 | 工作中 | 正在高强度榨取薪资 |
| 休息状态 | 休息中 | 摸鱼中 |
| 未配置 | 待配置 | null / unknown |
| 保存成功 | 已保存 | Config write success |
| 无变化 | 没有需要保存的修改 | No diff |
| 纯桌宠说明 | 开启后会隐藏任务栏入口，请确认托盘可用 | enable pure pet native flag |

幽默可以留给小猫动作，不放进关键设置和错误反馈里。

## 9. 实现边界

v0.4 UI polish 包含：

- Panel 折叠态改为“金额 + 短状态”。
- Panel 展开态信息层级优化。
- Panel 默认视觉统一为暖灰/墨色半透明风格。
- 设置窗口布局、间距、按钮层级、保存反馈统一。
- 设置窗口保持工具化，不可爱化。
- 原型和文档同步 UI polish 规格。

v0.4 UI polish 不包含：

- 主题系统。
- 主题切换入口。
- 多套皮肤。
- 设置页插画化。
- 大规模重写 Godot UI 框架。
- 用 UI 动效替代小猫动画素材。

## 10. 验收标准

### 10.1 文档验收

- `doc/v0.4-ui-polish-spec.md` 存在，并说明 v0.4 UI 定位。
- `doc/prototypes/prototype-spec.md` 引用本规格。
- 文档明确主题系统不进入 v0.4。
- 文档明确 Panel 折叠态为“金额 + 短状态”。
- 文档明确设置窗口保持工具化。

### 10.2 原型验收

- `doc/prototypes/index.html` 能展示 v0.4 UI polish 后的 Panel 折叠态和展开态。
- 原型能看出小猫是主角，Panel 是辅助薪资便签。
- 原型中设置窗口不出现可爱化主题包装。
- 原型中没有 Theme Tab 或外观切换入口。

### 10.3 实现验收

- 折叠态 Panel 显示金额和短状态，文字不溢出，垂直居中。
- 展开态 Panel 信息完整，层级清晰，不遮挡小猫主体。
- 设置窗口只有一个清晰的设置对话框，不出现窗口套窗口观感。

- 透明度、缩放、窗口模式、单双休等控件继续可用。
- UI polish 不破坏点击穿透、托盘找回、右键菜单、拖拽和宠物交互。
